## Codebase Patterns
- GDrive functions (_build_gdrive_client, _gdrive_ensure_folder, _gdrive_upload_bytes) live in app/services/legacy/daily_export.py and are importable from there
- Settings for GDrive: legacy_daily_google_drive_root_folder_id (root folder), legacy_daily_google_drive_id (shared drive ID, optional)
- Service account creds loaded via legacy_daily_google_service_account_json or legacy_daily_google_service_account_file
- All GDrive operations pass supportsAllDrives=True for shared drive compatibility
- Pre-existing mypy errors in legacy/engine.py, legacy/daily_export.py, and expenses/crud.py — ignore them, only check new files
- Python 3.12 available at system level (no venv); use `python3` not `python`
- Commit style: `feat: US-XXX - Title`
- Migrations go in `migrations/` folder as numbered SQL files (e.g., 005_*.sql)
- `_persist_batch_metadata` in `app/routers/expenses/_deps.py` is the central function for batch persistence
- `update_batch_gdrive_status` in `_deps.py` updates only gdrive_* fields on a batch
- GET /batches uses `select("*")` so new DB columns are auto-included in response
- Use `--follow-imports=skip` with mypy to check only changed files (pre-existing errors elsewhere)
- `snapshot_payload` in expense_batch_items is the single source of truth for re-download; fields map directly to `_build_xlsx` inputs
- Dashboard: `API_BASE` from `lib/supabase.ts`, auth via `X-Admin-Token` header with `TOKEN_KEY = 'lever-admin-token'` in sessionStorage
- Dashboard: `node_modules` may not be installed — run `npm install` before `tsc --noEmit`
- Dashboard: typecheck via `node_modules/.bin/tsc --noEmit` (not npx)
- Dashboard: useExpenses exports types `ExpenseStats`, `ExportResult`, `BatchRecord` for use in components
