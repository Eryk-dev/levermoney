## Codebase Patterns
- Migrations follow pattern `NNN_description.sql` with `CREATE TABLE IF NOT EXISTS` and `CREATE INDEX IF NOT EXISTS`
- Migration comments: descriptive header explaining purpose and idempotency guarantees
- Supabase project ID: `wrbrbhuhsaaupqsimkqz`
- Tests: 366 pytest tests, run with `python3 -m pytest`, all offline, ~2.4s
- Branch naming: `ralph/<feature-name>`
- `extrato_ingester.py` uses `_CHECK_PAYMENTS` sentinel + `_EXPENSE_TYPE_ABBREV` dict for composite keys
- New functions in `extrato_ingester.py` should be placed between `ingest_extrato_for_seller()` and `ingest_extrato_all_sellers()`
- The `ingest_extrato_from_csv()` function signature includes `month: str` (YYYY-MM) â€” not just seller+csv
- Supabase upsert: `db.table(...).upsert(row, on_conflict="col1,col2")` â€” comma-separated string
- Admin endpoints use `dependencies=[Depends(require_admin)]` at route level, not as function param
- `ingest_extrato_from_csv` returns `newly_ingested`/`skipped_internal` â€” map to DB cols `lines_ingested`/`lines_skipped`
- List endpoints use `{seller, count, data}` envelope pattern (see `expenses/crud.py`, `extrato.py`)
- `extrato_uploads` table uses `uploaded_at` (not `created_at`) and `month` (not `month_ref`)
- Dashboard components live flat in `dashboard/src/components/` (not in admin/ subdirectory)
- AdminPanel tabs use `styles.modeToggle` + `styles.modeBtn` / `styles.modeBtnActive` pattern
- Admin API auth token: `sessionStorage.getItem('lever-admin-token')` + header `X-Admin-Token`
- API base URL: `import { API_BASE } from '../lib/supabase'`
- Dashboard build: `cd dashboard && ./node_modules/.bin/tsc -b && ./node_modules/.bin/vite build` (npx tsc doesn't work)
- CSS Modules use `data-*` attribute selectors for status-variant badges (e.g. `data-status="completed"`)
- Adding new EVENT_TYPES requires updating 4 places in test_event_ledger.py: EXPECTED_TYPES, positive_types, negative_types, type_count
- CASH_TYPE_MAP, SKIP_TO_CASH_TYPE, SKIP_ABBREV live in `event_ledger.py` â€” import from there in tests and ingestion scripts
- Extrato lines can have amounts whose sign contradicts the mapped cash_type direction (e.g. "Pagamento com QR Pix" classified as income but with negative amount). Use `_align_cash_type_to_sign()` to correct.
- Same ref_id can appear multiple times in extrato with same classification (dispute pairs). Add sequence suffix to abbreviation to prevent idempotency key collisions.
