- ExpensesExportTab receives `sellers: Seller[]` and `onLogout` from AdminPanel; filter active sellers inside the component

- CSS uses CSS custom properties (--space-N, --text-xs, --ink, --ink-muted, --surface, --paper, --border) from global styles
- Dashboard uses CSS Modules (*.module.css), no Tailwind
- `dashboard_empresa` is the display name for sellers; fallback to `name` if not set
- PENDING_STATUS_FILTER = 'pending_review,auto_categorized' is the standard filter for pending expenses
- Export buttons should disable ALL export buttons while any export is in progress (use a single exportingSlug state)
- Confirmation modal pattern: store slug in state (null = closed), clear before triggering async action
- Polling pattern: use useRef for intervals/counts to avoid stale closures; stop polling on collapse, max retries, or condition met; cleanup all intervals on unmount
