## Codebase Patterns
- Expense lifecycle events use `record_expense_event()` with idempotency key `{seller}:{payment_id}:{event_type}` (3 parts)
- expense_captured uses sign "any" (positive=income, negative=expense), other expense events use "zero"
- All event_ledger queries (get_dre_summary, get_payment_statuses) already filter out `expense_*` events — same as `cash_*`
- Tests in `testes/test_event_ledger.py` cover pure functions; async tests use `unittest.mock.patch` on `record_event`
- Full test suite: 469 tests, ~4.2s
- Dual-write helper pattern: `_dual_write_expense_events()` as async helper, called after mp_expenses upsert, catches `EventRecordError` as warning
- Metadata for expense_captured must include ALL fields for Fase 3: expense_type, expense_direction, ca_category, auto_categorized, description, amount, date_created, date_approved, business_branch, operation_type, payment_method, external_reference, beneficiary_name, notes
- Signed amount: positive for income direction, negative for expense/transfer
- Ledger read helpers: `_fetch_expense_events()` → `_group_expense_events()` → `_build_expense_row()` pipeline reconstructs mp_expenses-compatible rows from event metadata
- Feature flag pattern for Fase 3: `settings.expenses_source` ('mp_expenses' | 'ledger') — consumers check flag and branch to old or new read path
- Router ledger migration: add `if settings.expenses_source == "ledger":` branch at top of each endpoint, import ledger helpers conditionally inside the block
- Ledger export: `get_pending_exports()` replaces mp_expenses query; `record_expense_event(event_type='expense_exported', signed_amount=0)` replaces status update
- Ledger rows have string `id` (reference_id); convert to int via `int(payment_id.split(':')[0])` for batch persistence (expense_batch_items.expense_id is bigint)
- Release report dual-write: `row["date"]` for both date_created/date_approved, `f"release_{desc_type}"` for operation_type, `payout_bank_account` for notes
- Export.py lazy imports (inside conditional blocks) must be patched at source (`app.services.event_ledger.record_expense_event`), not at module level
- Historical data backfill: INSERT INTO payment_events SELECT ... FROM mp_expenses ON CONFLICT DO NOTHING — safe to re-run, uses same idempotency_key pattern
- review_expense ledger mode: query payment_events for reference_id, write expense_reviewed event with review fields as metadata
