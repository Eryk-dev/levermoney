## Codebase Patterns
- Migrations use `IF NOT EXISTS` / `ADD COLUMN IF NOT EXISTS` for idempotency
- Migration files follow pattern `NNN_description.sql` in migrations/
- Test suite: 554 tests, all offline, ~2s via `python3 -m pytest`
- Frontend hooks follow `useExpenses.ts` pattern: TOKEN_KEY='lever-admin-token', authHeaders with X-Admin-Token, handleUnauthorized on 401
- Envelope responses (e.g. `{ seller, count, data }`) must extract `.data` in hooks
- Upload endpoints use `FormData` with `formData.append('files', file)` for multipart (no Content-Type header — browser sets it)
- `validate_extrato_coverage()` and `_months_range()` are public helpers in `extrato_ingester.py` for CSV coverage validation
- Extrato CSV dates: DD-MM-YYYY in file, YYYY-MM-DD after parsing by `_parse_account_statement()`
- Test fixtures for extrato: `sample_extrato_csv` in conftest.py, real files in `testes/data/extratos/`
- GDrive background upload pattern: `asyncio.create_task(asyncio.to_thread(sync_fn, ...))` — never block response
- `gdrive_client.py` has two upload functions: `upload_expenses_zip()` (DESPESAS) and `upload_extrato_csv()` (EXTRATOS)
- Multi-file upload endpoint: `POST /admin/sellers/{slug}/extrato/upload` — auto-detects months from CSV content
- Hidden file input pattern for card upload buttons: `<input ref={...} style=display:none multiple>` + `button.onClick → ref.click()`
- ExtratoTab uses `useExtrato` hook (not inline fetch) — follows same separation as ExpensesExportTab/useExpenses
- AdminPanel already had ExtratoTab wired in from US-007 commit — always check existing state before implementing
