# Data Model: Daily Cash Reconciliation

**Date**: 2026-04-10
**Feature**: [spec.md](spec.md)

## Entities

### Payment (ML API source)

Source: ML Payments API, fetched live via `ml_api.search_payments()`

| Field | Type | Meaning |
|-------|------|---------|
| id | int | Payment ID (unique) |
| status | string | approved, refunded, charged_back, cancelled, rejected |
| status_detail | string | accredited, by_admin, reimbursed, etc. |
| transaction_amount | float | Gross sale amount |
| shipping_amount | float | Amount buyer paid for shipping |
| net_received_amount | float | Net after all fees (via transaction_details) |
| date_approved | ISO datetime | When payment was confirmed (UTC-4) |
| money_release_date | ISO datetime | When ML releases funds (UTC-4) |
| money_release_status | string | "released" or "pending" |
| charges_details | array | Fee/shipping breakdown (source of truth) |
| order.id | string | Order ID (null for non-order payments) |
| collector.id | int | Present = seller is buying (skip) |
| description | string | "marketplace_shipment" = skip |
| refunds | array | Individual refund entries |

**Derived Values**:
- `competencia = _to_brt_date(date_approved)` — DRE date
- `cash_date = _to_brt_date(money_release_date)` — Baixa date
- `mp_fee = sum(charges[type=fee, from=collector, name!=financing_fee])`
- `shipping_cost = max(0, sum(charges[type=shipping, from=collector]) - shipping_amount)`
- `calculated_net = transaction_amount - mp_fee - shipping_cost`

### Extrato Line (ML CSV source)

Source: Account statement CSV, at `testes/data/extratos/`

| Field | Type | Meaning |
|-------|------|---------|
| date | string | Date in BRT (DD-MM-YYYY format) |
| transaction_type | string | "Liberacao de dinheiro", "DIFAL", etc. |
| reference_id | string | Payment ID or internal ML reference |
| amount | float | Signed amount (positive=credit, negative=debit) |
| balance | float | Running balance after this line |

**BR Number Format**: `1.234,56` (dot=thousands, comma=decimal)

### CA Entry (generated by system)

Three entries per approved payment:

| Entry Type | CA Account | Amount | Date Fields |
|------------|-----------|--------|-------------|
| Receita | contas-a-receber | transaction_amount | competencia, vencimento=cash_date |
| Comissao | contas-a-pagar | mp_fee | competencia, vencimento=cash_date |
| Frete | contas-a-pagar | shipping_cost | competencia, vencimento=cash_date |

### Daily Reconciliation (computed)

For each day D:

| Side | Formula |
|------|---------|
| CA total | sum(receita_baixas) - sum(comissao_baixas) - sum(frete_baixas) + sum(expense_entries) |
| Extrato total | sum(all extrato lines where date=D) |
| Divergence | abs(CA_total - extrato_total) |
| Pass criterion | divergence < R$0.05 |

## Relationships

```
Payment (1) ──generates──> (3) CA Entries [receita, comissao, frete]
Payment (1) ──matches───> (1) Extrato Line [Liberacao de dinheiro]
Payment (1) ──may have──> (N) Refund entries
Extrato Line (N) ──belongs to──> (1) Day
CA Entry (1) ──settles via──> (1) Baixa [on cash_date]
```

## State Transitions

### Payment Processing State

```
unprocessed → sale_approved → ca_sync_completed
                           → ca_sync_failed (retry)
sale_approved → refund_created → ca_sync_completed
```

### Reconciliation State (per day)

```
unchecked → tested → pass (divergence < R$0.05)
                   → fail (divergence >= R$0.05) → investigate
```
